---
title: 'Salvar Lead'
api: 'POST /rest/v1/rpc/save_event_lead'
description: 'Salva ou atualiza dados de um lead de evento'
---

## Objetivo

Persistir os dados do lead apos confirmacao ou recusa de participacao no evento. Cria novo registro ou atualiza existente baseado no telefone.

## Endpoint

```
POST https://tkrjfifrwttikwsghuju.supabase.co/rest/v1/rpc/save_event_lead
```

## Headers

```bash
apikey: SUA_ANON_KEY
Authorization: Bearer SUA_SERVICE_KEY
Content-Type: application/json
```

## Request Body

<ParamField body="p_contact_id" type="uuid" required>
  ID do contato no sistema
</ParamField>

<ParamField body="p_name" type="string" required>
  Nome do lead
</ParamField>

<ParamField body="p_phone_number" type="string" required>
  Telefone no formato internacional
</ParamField>

<ParamField body="p_confirmed_presence" type="boolean" required>
  `true` se confirmou presenca, `false` se recusou
</ParamField>

<ParamField body="p_email" type="string">
  Email do lead (opcional)
</ParamField>

<ParamField body="p_interested_future_events" type="boolean">
  `true` se quer saber de eventos futuros
</ParamField>

<ParamField body="p_language" type="string">
  Idioma: `pt-BR`, `en-US`, `es-ES`
</ParamField>

<ParamField body="p_invited_by" type="string">
  Nome de quem convidou o lead
</ParamField>

<ParamField body="p_evento_id" type="integer">
  ID do evento (obtido de `get_events_by_language`)
</ParamField>

<ParamField body="p_session_id" type="uuid">
  ID da sessao atual
</ParamField>

<ParamField body="p_chatwoot_conversation_id" type="integer">
  ID da conversa no Chatwoot
</ParamField>

<ParamField body="p_chatwoot_account_id" type="integer">
  ID da conta Chatwoot
</ParamField>

## Response - Sucesso

```json
{
  "success": true,
  "lead_id": 456,
  "message": "Lead salvo com sucesso"
}
```

## Response - Erro

```json
{
  "success": false,
  "error": "Campo p_name e obrigatorio"
}
```

## Comportamento Upsert

A funcao usa `ON CONFLICT` para atualizar se o telefone ja existe:

- **Novo lead:** Cria registro completo
- **Lead existente:** Atualiza apenas campos nao-nulos

<Info>
  Execute ANTES de enviar dados do Zoom ao lead, para garantir que o registro existe.
</Info>
